项目建议

做一个你工作里真的会用的小工具，例如需求澄清助手、竞品记录器、版本发布检查表、 研发周报生成器或客服反馈归因面板。目标是小而完整：有用户、场景、数据、页面、发布和复盘。

验收标准

  • 能用一句话讲清楚它解决谁的什么问题。
  • 有 GitHub 仓库、Issue、分支、PR 和提交记录。
  • 至少有一个 AI 参与的需求拆解、代码解释或排障记录。
  • 可以被别人打开试用，哪怕只是 GitHub Pages。
  • 有一次复盘：做对了什么、卡在哪里、下次怎么更快。

Cross-device entry

多端使用方式

现在就能用

  1. 电脑上用浏览器打开这个目录里的 index.html。
  2. 进度会保存在当前浏览器本地。
  3. 用左侧“导出”生成 JSON，在另一台设备点“导入”即可延续进度。

推荐入口

  1. 把本目录推到 GitHub 仓库。
  2. 开启 GitHub Pages，让它变成手机和电脑都能访问的网址。
  3. 后续如果需要真同步，可接 Supabase、GitHub Gist 或 Notion 数据库。

数据边界

这个版本不上传你的学习记录，登录门禁只在浏览器前端执行。它适合先跑通学习闭环； 当你确认自己会长期使用，再接入云端账号体系更稳。
