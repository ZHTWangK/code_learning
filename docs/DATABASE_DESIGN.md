# 全链路学习助手：数据库设计

版本：V0.1  
日期：2026-05-24

## 1. 设计结论

当前阶段采用 SQLite 作为本地服务端数据库，优先解决个人工具的真实认证、学习进度云端化雏形、资源沉淀、医疗信息化知识卡片、文档模板沉淀。

不建议一开始引入复杂多人组织、课程后台、权限审计和图谱能力。原因是当前最重要的是把个人使用闭环跑通，先保证数据能被结构化保存、能迁移、能继续演进。

## 2. 核心数据对象

| 对象 | 表名 | 说明 |
| --- | --- | --- |
| 用户 | `users` | 当前先支持单用户 owner，后续可扩展多用户 |
| 会话 | `sessions` | 服务端登录态，替代前端假门禁 |
| 学习进度 | `learning_progress` | 保存 12 周任务、笔记、里程碑等状态 JSON |
| 学习资源 | `learning_resources` | 记录文章、视频、课程、工具、案例、个人笔记 |
| 领域知识卡片 | `domain_cards` | 沉淀药品、费用等医疗信息化业务经验 |
| 文档模板 | `document_templates` | 沉淀 PRD、汇报、详细设计、会议纪要等模板 |

## 3. 当前取舍

### 3.1 进度状态先保留 JSON

`learning_progress.state_json` 暂时保存前端已有 localStorage 结构。这样可以低成本接入后端，不打断现有页面逻辑。

后续如果要做统计、趋势、周报分析，再把任务完成记录拆成 `task_completions`、`weekly_reviews` 等明细表。

### 3.2 资源和知识卡片先结构化

学习资源和领域知识库是长期资产，必须从一开始结构化建模。尤其是 `domain_cards`，字段覆盖背景、规则、对象、数据字段、状态流转、异常、产品化判断、验收标准和风险，便于后续沉淀药品、费用两个核心域。

### 3.3 认证先单用户

默认账号来自环境变量：

- `LEARNING_ASSISTANT_USER`
- `LEARNING_ASSISTANT_PASSWORD`

如果没有配置，默认使用 README 里的账号密码。生产或公网部署前必须改成强密码。

## 5. 本地数据库位置

默认数据库文件：

```text
data/learning-assistant.sqlite
```

该文件包含个人学习数据和登录信息，不建议提交到代码仓库。
